# TypeScript: `any` 타입

## any 타입이란?

기본 자료형 + 참조 자료형에 속한 **모든 값의 타입을 허용**하는 타입.

```typescript
const str: any = "hello";
const num: any = 30;
const bool: any = true;
const arr: any = [1, 2, 3];
const tuple: any = [1, "hello", true];
const obj: any = { name: "홍길동" };

function fn(a: any): any {
  return a;
}
```

→ 어떤 타입의 값을 할당해도 에러 없음.

---

## any 타입을 쓰면 안 되는 이유

타입이 불명확해져서 **TypeScript의 장점이 사라짐.**

```typescript
// ❌ any 사용 — 자동완성 안 됨
const user: any = { name: "홍길동", age: 20 };
user. // → 어떤 속성이 있는지 알 수 없음

// ✅ 타입 지정 — 자동완성 동작
const user: { name: string; age: number } = { name: "홍길동", age: 20 };
user. // → name, age 자동완성 제공
```

---

## 정리

|                | `any` | 명시적 타입        |
| -------------- | ----- | ------------------ |
| 모든 값 허용   | ✅    | ❌ (지정한 타입만) |
| 타입 안전성    | ❌    | ✅                 |
| 자동완성       | ❌    | ✅                 |
| 에러 조기 발견 | ❌    | ✅                 |

> **any는 값의 타입을 도저히 파악할 수 없는 경우에만 사용**  
> 그 외에는 되도록 사용하지 말 것
