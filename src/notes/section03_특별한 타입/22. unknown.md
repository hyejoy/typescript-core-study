# TypeScript: `unknown` 타입

## unknown 타입이란?

`any`처럼 **모든 값을 허용**하지만, 사용 시 반드시 **타입을 좁혀야** 하는 타입.

```typescript
const a: unknown = "hello";
const b: unknown = 30;
const c: unknown = true;
const d: unknown = [1, 2, 3];
// → 어떤 값을 할당해도 에러 없음
```

---

## any와의 차이

```typescript
// any — 그냥 사용 가능
const a: any = "hello";
const b: any = "world";
const c = a + b; // ✅ 에러 없음

// unknown — 타입을 좁혀야 사용 가능
const a: unknown = "hello";
const b: unknown = "world";
const c = a + b; // ❌ 에러 — unknown 타입은 바로 사용 불가
```

---

## 타입 좁히기 (Type Narrowing)

`typeof` 키워드 등으로 타입을 확인한 후 사용.

```typescript
const a: unknown = "hello";
const b: unknown = "world";

// if문으로 좁히기
if (typeof a === "string" && typeof b === "string") {
  const c = a + b; // ✅ string으로 좁혀졌으므로 에러 없음
  console.log(c); // "helloworld"
}

// 삼항 연산자로 좁히기
const c =
  typeof a === "string" && typeof b === "string" ? a + b : "문자열이 아님";
```

---

## any vs unknown 비교

|                | `any` | `unknown`             |
| -------------- | ----- | --------------------- |
| 모든 값 허용   | ✅    | ✅                    |
| 바로 사용 가능 | ✅    | ❌ (타입 좁히기 필요) |
| 타입 안전성    | ❌    | ✅                    |
| TS 취지에 부합 | ❌    | ✅                    |

---

## 결론

> `any`를 써야 할 상황이라면 **`unknown`을 쓰는 것이 더 TypeScript 취지에 맞음**  
> `unknown`은 타입 좁히기를 강제해서 타입 안전성을 유지하기 때문
