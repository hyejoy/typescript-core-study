# TypeScript: 객체 옵셔널 속성 & 타입 안전성

## 옵셔널 속성 복습

```typescript
const user: {
  name: string;
  age: number;
  getName?: () => string; // 있어도 되고 없어도 됨
} = {
  name: "홍길동",
  age: 10,
  // getName 없어도 에러 없음
};
```

---

## 주의사항

```typescript
// 옵셔널로 선언했다가 나중에 필수로 바꾸면 → 에러 발생
const user: {
  name: string;
  getName: () => string; // ? 제거
} = {
  name: "홍길동",
  // getName 없음 → ❌ 에러
};
```

---

## 옵셔널 속성과 타입 안전성

> **타입 안전성** = 타입의 일관성과 올바름을 유지해서 타입 관련 오류를 최소화하는 것

옵셔널 속성을 남용하면 타입 안전성이 떨어짐.

```typescript
// ❌ 전부 옵셔널 → 타입 안전성 낮음
const user: {
  name?: string;
  age?: number;
  getName?: () => string;
} = {}; // 빈 객체도 타입 에러 없음 → 의미 없는 타입
```

---

## 결론

|             | 일반 속성 | 옵셔널 속성 (`?`)  |
| ----------- | --------- | ------------------ |
| 반드시 포함 | ✅        | ❌                 |
| 타입 안전성 | 높음      | 낮음               |
| 사용 권장   | 기본값    | 진짜 선택적일 때만 |

> 옵셔널 속성은 **정말 필요한 경우에만 신중하게** 사용할 것  
> 무분별하게 사용하면 타입 지정의 의미가 없어짐
