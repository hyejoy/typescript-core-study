# TypeScript: 객체 옵셔널 속성 & 타입 안전성

## 옵셔널 속성 복습

```typescript
const user: {
  name: string;
  age: number;
  getName?: () => string; // 있어도 되고 없어도 됨
} = {
  name: "홍길동",
  age: 10,
  // getName 없어도 에러 없음
};
```

---

## 주의사항

```typescript
// 옵셔널로 선언했다가 나중에 필수로 바꾸면 → 에러 발생
const user: {
  name: string;
  getName: () => string; // ? 제거
} = {
  name: "홍길동",
  // getName 없음 → ❌ 에러
};
```

---

## 옵셔널 속성과 타입 안전성

> **타입 안전성** = 타입의 일관성과 올바름을 유지해서 타입 관련 오류를 최소화하는 것

옵셔널 속성을 남용하면 타입 안전성이 떨어짐.

```typescript
// ❌ 전부 옵셔널 → 타입 안전성 낮음
const user: {
  name?: string;
  age?: number;
  getName?: () => string;
} = {}; // 빈 객체도 타입 에러 없음 → 의미 없는 타입
```

---

## 옵셔널 속성이 타입 안전성을 떨어뜨리는 구체적 예시

```typescript
function greet(user: { name: string; age?: number }): void {
  console.log(`Hello, ${user.name}!`);
  console.log(`You are ${user.age} years old.`); // ⚠️ age가 없으면 undefined
}

// age 있음 → 정상
greet({ name: "철수", age: 20 }); // "Hello, 철수!", "You are 20 years old."

// age 없음 → 타입 에러는 없지만 런타임에서 문제 발생
greet({ name: "철수" }); // "Hello, 철수!", "You are undefined years old." ❌
```

### 왜 타입 에러가 안 나는가?

```
TypeScript는 매개변수의 타입 구조만 체크
→ age가 옵셔널이므로 없어도 타입 에러 없음
→ 하지만 실제 실행 시 age를 참조하면 undefined 발생
→ 타입 차원에서 잡을 수 없는 런타임 오류
```

## 결론

|             | 일반 속성 | 옵셔널 속성 (`?`)  |
| ----------- | --------- | ------------------ |
| 반드시 포함 | ✅        | ❌                 |
| 타입 안전성 | 높음      | 낮음               |
| 사용 권장   | 기본값    | 진짜 선택적일 때만 |

> 옵셔널 속성은 **정말 필요한 경우에만 신중하게** 사용할 것  
> 무분별하게 사용하면 타입 지정의 의미가 없어짐
