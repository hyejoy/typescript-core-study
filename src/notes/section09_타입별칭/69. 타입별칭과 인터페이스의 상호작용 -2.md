# TypeScript: 타입 별칭 (Type Alias)

## 타입 별칭이란?

타입에 **다른 이름(별칭)** 을 붙이는 것.  
`type` 키워드 사용.

```typescript
type 별칭 = 타입;
```

---

## 기본 예시

```typescript
// 기본 타입에 별칭 부여
type Email = string;
type Password = string;

const e: Email = "hong@example.com";
const p: Password = "1234";
```

---

## 타입 별칭의 장점

### 1. 의미 명확 전달

```typescript
// ❌ 의미 불분명
const e: string = "hong@example.com";

// ✅ 의미 명확
const e: Email = "hong@example.com";
```

### 2. 재사용성

```typescript
type Email = string;

const userEmail: Email = "user@example.com";
const adminEmail: Email = "admin@example.com"; // 동일한 별칭 재사용
```

### 3. 유지보수

```typescript
type Email = string; // → string | null 로 변경 시 한 곳만 수정하면 됨

const e1: Email = "hong@example.com";
const e2: Email = "lee@example.com";
// 모두 한 번에 반영됨
```

### 4. 확장성

```typescript
// 다양한 타입과 조합 가능
type ID = number | string;
type Status = "active" | "inactive" | "pending"; // 리터럴 타입
type User = { name: string; age: number }; // 객체 타입
```

---

## 인터페이스 vs 타입 별칭

|         | 인터페이스     | 타입 별칭             |
| ------- | -------------- | --------------------- |
| 키워드  | `interface`    | `type`                |
| 주 용도 | 객체 타입 정의 | 모든 타입에 이름 부여 |
| 병합    | ✅ 가능        | ❌ 불가               |
| 확장    | `extends`      | `&` (intersection)    |

> 앞으로 여러 타입 지정 방법과 조합하면서 타입 별칭의 매력을 더 살펴볼 예정

---

## 객체 타입 지정

인터페이스와 동일하게 객체 타입 지정 가능.

```typescript
// 인터페이스
interface Person {
  name: string;
  age: number;
  gender: string;
}

// 타입 별칭 — 동일한 역할
type Person = {
  name: string;
  age: number;
  gender: string;
};
```

---

## 객체 타입 문법 모두 사용 가능

### readonly + 옵셔널

```typescript
type Person = {
  readonly name: string; // 읽기 전용
  age: number;
  gender?: string; // 옵셔널
};
```

### 함수 시그니처

```typescript
// 인터페이스 방식
interface Add {
  (n1: number, n2: number): number;
}

// 타입 별칭 방식 — 화살표 함수 형태로 정의
type Add = (n1: number, n2: number) => number;

const add: Add = (n1, n2) => n1 + n2;
```

### 인덱스 시그니처

```typescript
// 인터페이스 방식
interface Dict {
  [key: string]: string;
}

// 타입 별칭 방식
type Dict = { [key: string]: string };
```

---

## 정리 — 객체 타입 관련 문법 사용 가능 여부

| 문법            | 인터페이스 |   타입 별칭   |
| --------------- | :--------: | :-----------: |
| readonly        |     ✅     |      ✅       |
| 옵셔널 (`?`)    |     ✅     |      ✅       |
| 함수 시그니처   |     ✅     |      ✅       |
| 인덱스 시그니처 |     ✅     |      ✅       |
| 병합            |     ✅     |      ❌       |
| extends 상속    |     ✅     | ❌ (`&` 사용) |

---

## 인터페이스와의 차이 — 병합 & 상속 불가

### 병합 ❌

```typescript
// 인터페이스 → 자동 병합 ✅
interface Person {
  name: string;
}
interface Person {
  age: number;
}
// → { name: string; age: number } 으로 합쳐짐

// 타입 별칭 → 에러 ❌
type Person = { name: string };
type Person = { age: number }; // 에러 — 중복 식별자
```

### 상속 ❌

```typescript
// 인터페이스 → extends 상속 ✅
interface Developer extends Person { skill: string; }

// 타입 별칭 → extends 불가 ❌
type Developer = extends Person { skill: string; } // 에러
```

---

## 타입 별칭에서 병합/상속 흉내내기 — intersection(`&`)

### 병합 흉내

```typescript
type PersonName = { name: string };
type PersonAge = { age: number };

// & 로 합치기
type User = PersonName & PersonAge;

const user: User = { name: "홍길동", age: 20 }; // ✅
```

### 상속 흉내

```typescript
type Animal = { name: string; age: number };

// extends 대신 & 사용
type Dog = Animal & { gender: string };

const dog: Dog = {
  name: "멍멍이",
  age: 3,
  gender: "male", // ✅
};
```

> ⚠️ `&` (intersection)으로 기능적으로 흉내낼 수 있지만  
> 인터페이스의 병합/상속과 **문법적으로 완전히 동일한 메커니즘은 아님**

---

## 인터페이스 + 타입 별칭 함께 쓰기

타입 별칭으로 복잡한 타입을 분리하면 인터페이스의 가독성과 재사용성이 높아짐.

### 패턴 1 — 리터럴 타입 분리

```typescript
// ❌ 인터페이스 안에 직접 리터럴 타입 나열
interface Person {
  name: string;
  gender: "male" | "female";
}

// ✅ 타입 별칭으로 분리
type Gender = "male" | "female";

interface Person {
  name: string;
  gender: Gender; // 훨씬 명확
}
```

### 패턴 2 — 상속 + 타입 별칭 재사용

```typescript
type Sound = "bark" | "meow" | "roar";

interface Animal {
  name: string;
  age: number;
}

interface Pet extends Animal {
  sound: Sound; // 타입 별칭 재사용
}

interface WildAnimal extends Animal {
  sound: Sound; // 동일한 타입 별칭 재사용
  habitat: string;
}

// 함수에서도 재사용
function processAnimalSound(sound: Sound): void {
  console.log(sound);
}
```

### 패턴 3 — 반복되는 함수 타입 분리

```typescript
// ❌ 동일한 함수 타입이 반복됨
interface Calculator {
  add: (n1: number, n2: number) => number;
  subtract: (n1: number, n2: number) => number;
}

// ✅ 타입 별칭으로 분리 → 재사용 + 가독성 UP
type MathOperation = (n1: number, n2: number) => number;

interface Calculator {
  add: MathOperation;
  subtract: MathOperation;
}
```

> 타입 별칭 변경 시 사용하는 모든 곳에 자동 반영 → **유지보수 용이**

---

## 타입 별칭의 재료로 인터페이스 사용하기

### 패턴 4 — 인터페이스를 union으로 묶기

```typescript
interface Dog {
  name: string;
  breed: string;
}
interface Cat {
  name: string;
  indoor: boolean;
}

// 두 인터페이스를 union으로 묶어 범용적인 타입 생성
type Pet = Dog | Cat;

const pet1: Pet = { name: "멍멍이", breed: "골든리트리버" }; // Dog ✅
const pet2: Pet = { name: "야옹이", indoor: true }; // Cat ✅
```

> 두 인터페이스의 공통 분모가 없어서 `extends`로 확장하기 애매할 때  
> 타입 별칭 + union으로 묶으면 범용적인 타입 생성 가능

### 패턴 5 — 인터페이스를 intersection으로 확장하기

```typescript
interface BasicAddress {
  street: string;
  city: string;
}

// BasicAddress를 재료로 더 확장된 타입 생성
type FullAddress = BasicAddress & {
  country: string;
  zipCode: string;
};

const address: FullAddress = {
  street: "강남대로",
  city: "서울",
  country: "한국",
  zipCode: "06000",
}; // ✅
```

> 인터페이스의 속성을 그대로 가져오면서 새 속성을 추가할 때 유용

---

## 인터페이스 ↔ 타입 별칭 상호작용 패턴 총정리

| 방향                        | 패턴                  | 예시                                                      |
| --------------------------- | --------------------- | --------------------------------------------------------- |
| 타입 별칭 → 인터페이스 재료 | 리터럴 타입 분리      | `type Gender = "male" \| "female"`                        |
| 타입 별칭 → 인터페이스 재료 | 반복 함수 타입 분리   | `type MathOperation = (n1: number, n2: number) => number` |
| 인터페이스 → 타입 별칭 재료 | union으로 묶기        | `type Pet = Dog \| Cat`                                   |
| 인터페이스 → 타입 별칭 재료 | intersection으로 확장 | `type FullAddress = BasicAddress & { ... }`               |
