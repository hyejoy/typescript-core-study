# TypeScript: 타입 별칭 (Type Alias)

## 타입 별칭이란?

타입에 **다른 이름(별칭)** 을 붙이는 것.  
`type` 키워드 사용.

```typescript
type 별칭 = 타입;
```

---

## 기본 예시

```typescript
// 기본 타입에 별칭 부여
type Email = string;
type Password = string;

const e: Email = "hong@example.com";
const p: Password = "1234";
```

---

## 타입 별칭의 장점

### 1. 의미 명확 전달

```typescript
// ❌ 의미 불분명
const e: string = "hong@example.com";

// ✅ 의미 명확
const e: Email = "hong@example.com";
```

### 2. 재사용성

```typescript
type Email = string;

const userEmail: Email = "user@example.com";
const adminEmail: Email = "admin@example.com"; // 동일한 별칭 재사용
```

### 3. 유지보수

```typescript
type Email = string; // → string | null 로 변경 시 한 곳만 수정하면 됨

const e1: Email = "hong@example.com";
const e2: Email = "lee@example.com";
// 모두 한 번에 반영됨
```

### 4. 확장성

```typescript
// 다양한 타입과 조합 가능
type ID = number | string;
type Status = "active" | "inactive" | "pending"; // 리터럴 타입
type User = { name: string; age: number }; // 객체 타입
```

---

## 인터페이스 vs 타입 별칭

|         | 인터페이스     | 타입 별칭             |
| ------- | -------------- | --------------------- |
| 키워드  | `interface`    | `type`                |
| 주 용도 | 객체 타입 정의 | 모든 타입에 이름 부여 |
| 병합    | ✅ 가능        | ❌ 불가               |
| 확장    | `extends`      | `&` (intersection)    |

> 앞으로 여러 타입 지정 방법과 조합하면서 타입 별칭의 매력을 더 살펴볼 예정
