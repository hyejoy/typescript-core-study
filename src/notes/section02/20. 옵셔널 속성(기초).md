# TypeScript: 옵셔널 속성 (Optional Property)

## JS vs TS — 매개변수 개수 체크

```javascript
// JavaScript — 매개변수 개수 안 맞아도 에러 없음
function printName(lastName, firstName) {
  console.log(lastName + firstName);
}
printName("홍"); // 에러 없음
```

```typescript
// TypeScript — 매개변수 개수까지 정확하게 체크
function printName(lastName: string, firstName: string): string {
  return lastName + firstName;
}
printName("홍"); // ❌ 에러 — firstName 값이 없음
```

---

## 옵셔널 속성 (`?`)

매개변수나 속성을 **선택적**으로 만드는 문법.  
값을 전달해도 되고, 전달하지 않아도 됨.

```typescript
// firstName? → 선택적 매개변수
function printName(lastName: string, firstName?: string): string {
  return lastName + (firstName ?? "");
}

printName("홍"); // ✅ OK
printName("홍", "길동"); // ✅ OK
```

---

## 객체 속성에 옵셔널 사용

```typescript
// gender를 나중에 추가할 수 있도록 선택적으로 지정
const user: {
  name: string;
  age: number;
  gender?: string; // 있어도 되고 없어도 됨
} = {
  name: "홍길동",
  age: 20,
};

user.gender = "male"; // ✅ OK — 옵셔널로 지정했기 때문
```

---

## TS가 동적 속성 추가를 막는 이유

```typescript
const user: { name: string; age: number } = { name: "홍길동", age: 20 };

user.gender = "male"; // ❌ 에러 — gender는 타입에 없음
```

타입에 명시되지 않은 속성은 추가 불가 → 미리 `?`로 선언해야 함.

---

## 정리

|              | 일반 속성      | 옵셔널 속성 (`?`)     |
| ------------ | -------------- | --------------------- |
| 문법         | `name: string` | `name?: string`       |
| 값 필수 여부 | ✅ 필수        | ❌ 선택적             |
| 없을 때 타입 | —              | `string \| undefined` |

> 옵셔널 속성은 이후 여러 개념에서 자주 등장하므로 잘 기억해둘 것
