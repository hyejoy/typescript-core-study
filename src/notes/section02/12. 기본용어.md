# TypeScript 입문: JavaScript 기본 용어 복습

## TypeScript란?

JavaScript의 **값(value)에 타입을 명확하게 지정**해주는 언어.

```
JavaScript: 값이 어떤 타입인지 신경 쓰지 않음
TypeScript: 값의 타입을 명시적으로 선언하고 관리
```

---

## JS 코드 한 줄에서 보는 개념들

```javascript
const num = 10 + 20;
```

| 요소         | 예시      | 설명                                           |
| ------------ | --------- | ---------------------------------------------- |
| **키워드**   | `const`   | JS 문법에서 예약된 단어. 다른 용도로 사용 불가 |
| **식별자**   | `num`     | 특정 코드를 식별하는 이름                      |
| **변수명**   | `num`     | `const` 키워드와 함께 쓰인 식별자              |
| **연산자**   | `=`, `+`  | 연산을 수행하는 기호                           |
| **피연산자** | `10 + 20` | 연산자의 대상 (연산자 관점에서 부르는 명칭)    |
| **표현식**   | `10 + 20` | 값으로 도출될 수 있는 식                       |
| **값**       | `30`      | 표현식의 결과                                  |

---

## 키워드와 식별자의 관계

키워드의 역할에 따라 식별자를 다르게 부름.

```javascript
const num = 30; // num → 변수명
function greet() {} // greet → 함수명
class User {} // User → 클래스명
```

---

## 표현식 vs 값

```javascript
const num = 10 + 20;
//          ↑           ↑
//        표현식        값 (30)
//      (계산되기 전)  (이미 계산 완료)
```

> 변수에는 표현식 자체가 저장되는 게 아니라 **계산이 끝난 값(30)** 이 저장됨

---

## TypeScript를 배우기 전 핵심 포인트

```
JavaScript의 값  →  TypeScript에서 타입을 명시

const num = 30;          // JS: 그냥 30
const num: number = 30;  // TS: 30이고, 타입은 number야
```

앞으로 TypeScript는 이 **값이 어떤 타입인지**를 명확하게 다루는 것이 핵심.
