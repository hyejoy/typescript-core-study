# TypeScript: 클래스 타입 지정

## JavaScript 클래스 → TypeScript로 변환 시

JS 클래스를 TS로 그대로 가져오면 타입 에러 발생.  
→ 클래스에서 초기화하는 **모든 속성의 타입을 명시**해야 함.

---

## 문법

```typescript
class 클래스명 {
  // 1. 클래스 키워드 바로 아래 속성 타입 선언
  속성명: 타입;

  // 2. constructor 매개변수 타입 지정
  constructor(속성명: 타입) {
    this.속성명 = 속성명;
  }

  // 3. 메서드 타입 지정 (함수 시그니처)
  메서드명(매개변수: 타입): 반환타입 { ... }
}
```

---

## 예시

```typescript
class Person {
  // 속성 타입 선언
  name: string;
  age: number;

  // constructor 타입 지정
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  // 메서드 타입 지정
  updateName(name: string): void {
    this.name = name;
    console.log(`이름이 ${name}으로 변경되었습니다.`);
  }
}

const person1 = new Person("철수", 20);
console.log(person1); // { name: "철수", age: 20 }

person1.updateName("가영");
console.log(person1); // { name: "가영", age: 20 }
```

## 핵심 규칙

| 위치                 | 타입 지정 방법                          |
| -------------------- | --------------------------------------- |
| 클래스 속성          | 클래스 선언부 바로 아래 `속성명: 타입`  |
| constructor 매개변수 | 함수 시그니처와 동일 `(매개변수: 타입)` |
| 메서드               | 매개변수 + 반환값 타입 지정             |

> 클래스에서 다루는 **모든 속성의 타입을 명시**해야 타입 에러 없음

---

## implements — 인터페이스/타입 별칭 구현 강제

`implements` 키워드로 클래스가 특정 타입을 구현하도록 강제.

```typescript
interface Animal {
  name: string;
  makeSound(): void;
}

// Dog는 Animal 인터페이스를 구현해야 함
class Dog implements Animal {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  makeSound(): void {
    console.log("멍멍!");
  }
}
```

- `name`과 `makeSound`를 구현하지 않으면 → ❌ 에러
- 타입 별칭도 동일하게 사용 가능

```typescript
type Animal = { name: string; makeSound(): void; };
class Cat implements Animal { ... } // ✅ 타입 별칭도 OK
```

## implements vs 추상 클래스 비교

|                | `implements`           | 추상 클래스              |
| -------------- | ---------------------- | ------------------------ |
| 기반           | 인터페이스 / 타입 별칭 | 추상 클래스              |
| 인스턴스 생성  | ✅ 클래스는 가능       | ❌ 추상 클래스 직접 불가 |
| 구현 강제      | ✅                     | ✅                       |
| 기본 구현 제공 | ❌                     | ✅ (일반 메서드로)       |
| 상속           | `extends` 별도 사용    | `extends`로 상속         |

---

## implements vs 추상 클래스 상세 비교

### 차이점 1 — 속성 구현 방식

```typescript
// implements — super() 불필요
class Dog implements Animal {
  name: string;
  constructor(name: string) {
    this.name = name; // super 호출 필요 없음
  }
}

// 추상 클래스 — super() 필수
class Dog extends AbstractAnimal {
  name: string;
  constructor(name: string) {
    super(); // 부모 constructor 호출 필수
    this.name = name;
  }
}
```

### 차이점 2 — 기본 메서드 구현 제공

```typescript
// implements — 기본 구현 제공 불가 (타입만 정의)
interface Animal {
  makeSound(): void; // 구현 불가, 타입만 정의
}

// 추상 클래스 — 기본 구현 제공 가능
abstract class AbstractAnimal {
  makeSound(): void {
    console.log("..."); // 기본 구현 제공 가능
  }
}
```

### 차이점 3 — 다중 구현/상속

```typescript
// implements — 여러 타입 동시 구현 가능
interface Animal { name: string; makeSound(): void; }
type Runnable = { run(): void; };

class Dog implements Animal, Runnable { // ✅ 여러 개 가능
  name: string;
  makeSound(): void { console.log("멍!"); }
  run(): void { console.log("달린다!"); }
}

// 추상 클래스 — 단일 상속만 가능
class Dog extends AbstractAnimal { ... }       // ✅
class Dog extends A extends B { ... }          // ❌ 불가
```

## 언제 무엇을 쓸까?

| 상황                                           | 권장         |
| ---------------------------------------------- | ------------ |
| 여러 타입을 동시에 강제해야 할 때              | `implements` |
| 기본 동작을 제공하면서 추가 구현을 강제할 때   | 추상 클래스  |
| 인터페이스/타입 별칭 기반으로 타입만 강제할 때 | `implements` |
| 공통 로직을 부모 클래스에서 제공하고 싶을 때   | 추상 클래스  |

## implements 대상 타입의 다양한 형태

### 상속된 인터페이스 implements

```typescript
interface Animal {
  name: string;
  makeSound(): void;
}
interface Runnable extends Animal {
  run(): void;
}

// Runnable이 Animal을 상속 → Animal의 속성도 구현 강제
class Dog implements Runnable {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
  makeSound(): void {
    console.log("멍!");
  }
  run(): void {
    console.log("달린다!");
  }
}
```

### intersection 타입 별칭 implements

```typescript
type Animal = { name: string; makeSound(): void; };
type Runnable = Animal & { run(): void };

class Dog implements Runnable {
  name: string;
  makeSound(): void { ... }
  run(): void { ... }
}
```

### 옵셔널 속성 → 구현 안 해도 OK

```typescript
interface Animal {
  name?: string; // 옵셔널
  makeSound(): void;
}

class Dog implements Animal {
  // name 없어도 에러 없음 ✅
  makeSound(): void {
    console.log("멍!");
  }
}
```

### 인덱스 시그니처 implements

```typescript
interface Dictionary {
  [key: string]: string;
}

class WordBook implements Dictionary {
  [key: string]: string;

  constructor(word: string, definition: string) {
    this[word] = definition;
  }
}

const book = new WordBook("dog", "강아지");
book.dog; // "강아지"
```

### ⚠️ readonly는 implements 구현 강제 대상 예외

```typescript
interface Animal {
  readonly name: string; // readonly 지정
}

class Dog implements Animal {
  name: string; // readonly 없어도 에러 없음

  constructor(name: string) {
    this.name = name;
  }
}

const dog = new Dog("멍멍이");
dog.name = "바둑이"; // ✅ 변경 가능 — readonly가 강제되지 않음
```

> `readonly`는 implements 구현 대상에서 **예외** — 서브 클래스에서 직접 붙여야 효과 있음
