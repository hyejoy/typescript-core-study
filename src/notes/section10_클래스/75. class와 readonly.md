# TypeScript: `readonly` 키워드

## readonly란?

객체 타입의 속성에 `readonly`를 붙이면 **해당 속성값을 수정 불가능**하게 만드는 키워드.

---

## 사용법

```typescript
const user: {
  name: string;
  readonly age: number; // age는 읽기 전용
} = {
  name: "홍길동",
  age: 20,
};

user.name = "철수"; // ✅ name은 수정 가능
user.age = 30; // ❌ 에러 — age는 readonly 프로퍼티라 할당 불가
```

---

## 인터페이스에서도 사용 가능

```typescript
interface User {
  name: string;
  age: number;
  readonly gender: string; // 읽기 전용
}

const user: User = { name: "홍길동", age: 20, gender: "male" };

user.name = "이철수"; // ✅ 수정 가능
user.gender = "female"; // ❌ 에러 — readonly property
```

---

## 정리

|      | 일반 속성     | `readonly` 속성       |
| ---- | ------------- | --------------------- |
| 읽기 | ✅            | ✅                    |
| 수정 | ✅            | ❌                    |
| 용도 | 일반적인 속성 | 변경되면 안 되는 속성 |

> `readonly`는 인터페이스뿐만 아니라 **객체 타입 어디서든 사용 가능**

---

## 클래스에서 readonly

```typescript
class User {
  readonly name: string;

  constructor(name: string) {
    this.name = name;
  }
}

const user = new User("철수");
user.name = "영희"; // ❌ 에러 — readonly 속성은 재할당 불가
```

> `public`, `private`, `protected`와 함께 사용 가능  
> ex) `private readonly name: string`

---

## ⚠️ 참조 자료형에서 readonly 주의

`readonly`는 **재할당 불가** 이지 **내부 값 변경 불가**가 아님.

```typescript
class Example {
  readonly numbers: number[] = [1, 2, 3];
}

const ex = new Example();

// 내부 값 변경 → ✅ 가능 (참조 자료형 특성)
ex.numbers[0] = 20;
console.log(ex.numbers); // [20, 2, 3]

// 재할당 → ❌ 불가
ex.numbers = [4, 5, 6]; // 에러 — readonly
```

## 추상 클래스에서 readonly — 강제되지 않음

```typescript
abstract class Animal {
  abstract readonly name: string; // readonly를 추상 속성에 붙여도
}

class Dog extends Animal {
  name: string; // readonly 없어도 타입 에러 없음 → 강제 아님

  constructor(name: string) {
    super();
    this.name = name;
  }
}

const dog = new Dog("멍멍이");
dog.name = "바둑이"; // ✅ readonly가 강제되지 않아서 변경 가능
```

→ 실제로 변경 불가하게 하려면 **서브 클래스에서 직접** `readonly` 붙여야 함

```typescript
class Dog extends Animal {
  readonly name: string; // 직접 readonly 지정해야 효과 있음
  constructor(name: string) {
    super();
    this.name = name;
  }
}
```
