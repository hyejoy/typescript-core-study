# TypeScript: 추상 클래스 (Abstract Class)

## 추상 클래스란?

- **직접 인스턴스 생성 불가**
- 서브 클래스가 반드시 구현해야 할 속성/메서드를 **강제**하는 용도
- `abstract` 키워드로 선언 (JS에는 없는 TS 전용 개념)

---

## 문법

```typescript
abstract class 클래스명 {
  abstract 속성명: 타입; // 추상 속성
  abstract 메서드명(): 반환타입; // 추상 메서드
}
```

---

## 예시

```typescript
abstract class Animal {
  abstract name: string; // 추상 속성 — 서브 클래스에서 구현 강제
  abstract makeSound(): void; // 추상 메서드 — 서브 클래스에서 구현 강제
}

// ❌ 추상 클래스는 직접 인스턴스 생성 불가
const animal = new Animal(); // 에러

// ✅ 서브 클래스에서 상속 후 구현
class Dog extends Animal {
  name: string;

  constructor(name: string) {
    super();
    this.name = name;
  }

  makeSound(): void {
    console.log("멍멍!");
  }
}

class Cat extends Animal {
  name: string;

  constructor(name: string) {
    super();
    this.name = name;
  }

  makeSound(): void {
    console.log("야옹!");
  }
}

const dog = new Dog("멍멍이");
dog.name; // "멍멍이"
dog.makeSound(); // "멍멍!"

const cat = new Cat("야옹이");
cat.makeSound(); // "야옹!"
```

---

## 추상 속성/메서드를 구현하지 않으면?

```typescript
class Dog extends Animal {
  // name과 makeSound를 구현하지 않으면 ❌ 에러
  // → 추상 클래스의 속성/메서드 구현은 의무
}
```

---

## 추상 클래스 vs 일반 클래스

|                       | 일반 클래스 | 추상 클래스      |
| --------------------- | ----------- | ---------------- |
| 인스턴스 생성         | ✅ 가능     | ❌ 불가          |
| 상속                  | ✅          | ✅               |
| 추상 속성/메서드      | ❌          | ✅               |
| 서브 클래스 구현 강제 | ❌          | ✅               |
| 키워드                | `class`     | `abstract class` |

---

## 언제 쓰나?

여러 서브 클래스가 **공통적으로 가져야 할 속성/메서드를 일정하게 강제**하고 싶을 때.

```
Animal (추상 클래스)
  ├── Dog (name, makeSound 구현 강제)
  ├── Cat (name, makeSound 구현 강제)
  └── Bird (name, makeSound 구현 강제)
```

---

## 추상 클래스에서 직접 구현도 가능

추상 클래스는 추상 속성/메서드로 **강제**하는 것 외에도  
일반 속성/메서드를 직접 구현하여 **상속 클래스로 활용**할 수 있음.

```typescript
abstract class Animal {
  // abstract 제거 → 직접 구현
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  makeSound(): void {
    console.log("...");
  }
}

// 강제 구현 없어도 OK — 단순 상속 클래스로 동작
class Dog extends Animal {
  constructor(name: string) {
    super(name); // 부모 constructor 호출
  }
}

const dog = new Dog("멍멍이");
dog.name; // "멍멍이" ✅
dog.makeSound(); // "..." ✅
```

## abstract + 접근 제어자 함께 사용

```typescript
abstract class Animal {
  protected name: string;  // 상속받은 클래스까지만 접근 가능

  constructor(name: string) {
    this.name = name;
  }

  protected makeSound(): void { ... }
}

class Dog extends Animal {
  constructor(name: string) { super(name); }
}

const dog = new Dog("멍멍이");
dog.name;        // ❌ 에러 — protected는 외부 접근 불가
dog.makeSound(); // ❌ 에러 — protected는 외부 접근 불가
```

> 추상 클래스는 직접 인스턴스를 생성할 수 없으므로  
> `private` / `protected` 사용 시 활용 범위가 일반 클래스보다 더 제한됨

---

## 추상 속성/메서드 vs 일반 속성/메서드 in 추상 클래스

|                       | 추상 속성/메서드 (`abstract`) | 일반 속성/메서드      |
| --------------------- | ----------------------------- | --------------------- |
| 서브 클래스 구현 강제 | ✅                            | ❌                    |
| 직접 구현             | ❌                            | ✅                    |
| 인스턴스 생성         | ❌ (추상 클래스 공통)         | ❌ (추상 클래스 공통) |
