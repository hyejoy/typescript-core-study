# TypeScript: 접근 제어자 (Access Modifier)

## 접근 제어자란?

클래스의 속성과 메서드에 대한 **접근 범위를 제어**하는 키워드.

| 접근 제어자 | 클래스 내부 | 상속받은 클래스 | 클래스 외부 |
| ----------- | :---------: | :-------------: | :---------: |
| `public`    |     ✅      |       ✅        |     ✅      |
| `protected` |     ✅      |       ✅        |     ❌      |
| `private`   |     ✅      |       ❌        |     ❌      |

> 접근 제어자를 명시하지 않으면 기본값은 `public`

---

## public

```typescript
class Person {
  public name: string; // 어디서든 접근 가능 (기본값)
  public age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

const person = new Person("홍길동", 20);
person.name = "이철수"; // ✅ 외부에서 접근/수정 가능
```

---

## private

```typescript
class Person {
  private name: string; // 클래스 내부에서만 접근 가능
  public age: number;

  constructor(name: string, age: number) {
    this.name = name; // ✅ 내부 접근 OK
    this.age = age;
  }

  updateName(name: string): void {
    this.name = name; // ✅ 내부 접근 OK
  }
}

const person = new Person("홍길동", 20);
person.name = "이철수"; // ❌ 에러 — private 속성은 외부 접근 불가
person.updateName("이철수"); // ✅ 메서드를 통해서만 변경 가능
```

---

## protected

```typescript
class Person {
  public name: string;
  public age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  protected updateName(name: string): void {
    // 상속받은 클래스까지만 접근 가능
    this.name = name;
  }
}

class Employee extends Person {
  private employeeId: number;

  constructor(name: string, age: number, employeeId: number) {
    super(name, age);
    this.employeeId = employeeId;
  }

  updateEmployeeName(name: string): void {
    this.updateName(name); // ✅ 상속받은 클래스에서 protected 메서드 접근 가능
  }
}

const employee = new Employee("홍길동", 20, 1);
employee.updateEmployeeName("이순신"); // ✅
employee.updateName("이순신"); // ❌ 외부에서는 protected 접근 불가
```

---

## private vs protected 핵심 차이

```
private   → 자기 클래스에서만 접근 가능 (상속받은 클래스도 ❌)
protected → 자기 클래스 + 상속받은 클래스까지 접근 가능 (외부 ❌)
```

---

## 접근 제어자를 쓰는 이유

```typescript
// 접근 제어자 없을 때 → 외부에서 직접 속성 변경 가능 (의도치 않은 수정 위험)
person.name = "아무개"; // 누구나 바꿀 수 있음

// private + 메서드로 제어 → 정해진 방식으로만 변경 가능
person.updateName("아무개"); // 메서드 통해서만 변경 가능 → 안전
```
