# TypeScript: 인터페이스로 함수 타입 지정

## JS에서 함수도 객체 → 인터페이스로 함수 타입 지정 가능

---

## 인터페이스 함수 타입 문법

```typescript
interface 인터페이스명 {
  (매개변수: 타입): 반환타입;
}
```

---

## 예시

```typescript
// 인터페이스 정의
interface Add {
  (n1: number, n2: number): number;
}

interface Greet {
  (name: string): string;
}

// 인터페이스 타입 사용 (함수 표현식에만 적용 가능)
const add: Add = (n1, n2) => n1 + n2;
const greet: Greet = (name) => `Hello, ${name}!`;
```

---

## 인터페이스 vs 기존 방식 비교

```typescript
// 기존 방식 1 — 함수에 직접
const add = (n1: number, n2: number): number => n1 + n2;

// 기존 방식 2 — 변수에 타입
const add: (n1: number, n2: number) => number = (n1, n2) => n1 + n2;

// 인터페이스 방식
interface Add {
  (n1: number, n2: number): number;
}
const add: Add = (n1, n2) => n1 + n2;
```

→ 결과는 동일, 인터페이스 방식이 가독성 더 좋음

---

## ⚠️ 제약사항 — 함수 선언문에는 사용 불가

```typescript
// ✅ 함수 표현식 → 인터페이스 타입 사용 가능
const add: Add = (n1, n2) => n1 + n2;

// ❌ 함수 선언문 → 인터페이스 타입 사용 불가
function add(n1, n2): Add { ... } // 이런 방법 없음

// 함수 선언문은 기존 방식으로만 타입 지정
function add(n1: number, n2: number): number {
  return n1 + n2;
}
```

---

## 정리

|                      | 함수 선언문               | 함수 표현식             |
| -------------------- | ------------------------- | ----------------------- |
| 인터페이스 타입 지정 | ❌ 불가                   | ✅ 가능                 |
| 타입 지정 방법       | 매개변수/반환값 직접 명시 | 인터페이스 or 직접 명시 |
