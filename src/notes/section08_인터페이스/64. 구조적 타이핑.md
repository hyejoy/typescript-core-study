# TypeScript: 구조적 타이핑 (Structural Typing)

## 구조적 타이핑이란?

객체의 **형태(구조)** 에 기반하여 타입을 검사하는 시스템.  
인터페이스 이름이 달라도 **키와 값의 타입이 동일하면 같은 타입으로 인식**.

---

## 예시

```typescript
interface User {
  name: string;
  age: number;
}

interface Person {
  name: string;
  age: number;
}

const user: User = { name: "홍길동", age: 20 };
const person: Person = { name: "이철수", age: 25 };

// 인터페이스 이름이 달라도 구조가 같으면 할당 가능
user = person; // ✅
person = user; // ✅
```

---

## 구조적 타이핑이 허용되지 않는 경우

### 값의 타입이 다를 때

```typescript
interface User {
  name: string;
  age: number;
}
interface Person {
  name: string;
  age: string;
} // age 타입 다름

user = person; // ❌ 에러 — age 타입 불일치
```

### 옵셔널 속성이 붙을 때

```typescript
interface User {
  name: string;
  age: number;
}
interface Person {
  name: string;
  age?: number;
} // 옵셔널

user = person; // ❌ 에러 — 형태가 다름 (age가 없을 수도 있음)
```

### readonly는 OK

```typescript
interface User {
  name: string;
  age: number;
}
interface Person {
  readonly name: string;
  age: number;
} // readonly

user = person; // ✅ — readonly는 형태를 바꾸지 않음
```

---

## 함수 매개변수에서도 적용

```typescript
interface User {
  name: string;
  age: number;
}

function greet(user: User): void {
  console.log(user.name);
}

// 타입 명시 없이 추론된 객체도 구조가 같으면 OK
const user1 = { name: "홍길동", age: 20 }; // → { name: string; age: number }으로 추론
greet(user1); // ✅ — User 인터페이스와 구조가 같으므로 통과
```

---

## 정리

| 조건                   | 구조적 타이핑 허용 여부 |
| ---------------------- | :---------------------: |
| 키와 값 타입 모두 일치 |           ✅            |
| 값 타입 하나라도 다름  |           ❌            |
| 옵셔널 속성 차이 있음  |           ❌            |
| readonly 차이만 있음   |           ✅            |

> TypeScript는 **이름이 아닌 구조로 타입을 판단**  
> 출처(인터페이스 이름)가 달라도 형태만 같으면 같은 타입으로 인식
