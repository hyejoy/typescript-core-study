# TypeScript: 인터페이스 병합 (Interface Merging)

## 인터페이스 병합이란?

**같은 이름의 인터페이스**가 여러 개 선언되면 자동으로 **하나로 합쳐지는** 현상.  
타입 에러가 발생하지 않음.

---

## 예시

```typescript
interface User {
  name: string;
}

interface User {
  age: number;
}

interface User {
  gender: string;
}

// 결과 — 세 인터페이스가 하나로 병합됨
// interface User {
//   name: string;
//   age: number;
//   gender: string;
// }

const user: User = {
  name: "홍길동", // ✅
  age: 20, // ✅
  gender: "male", // ✅ 세 속성 모두 필요
};

const user2: User = { name: "홍길동" }; // ❌ age, gender 없으므로 에러
```

---

## 위치와 무관하게 병합됨

```typescript
// 변수 선언이 중간에 있어도 전체 병합 결과로 타입 체크
interface User {
  name: string;
}

const user: User = { name: "홍길동" }; // gender 없어도 될 것 같지만 ❌ 에러

interface User {
  gender: string;
}
```

> TypeScript 컴파일러는 **인터페이스 병합을 먼저 수행한 후** 타입 검사를 진행함  
> → 위치 상관없이 같은 이름의 인터페이스는 모두 병합

---

## ⚠️ 대규모 프로젝트에서 주의

```
여러 파일에서 같은 이름의 인터페이스 선언
→ 의도치 않은 병합 발생
→ 예상치 못한 타입 에러 발생 가능
```

> 인터페이스 이름이 중복되면 병합된다는 사실을 알아야  
> 대규모 프로젝트에서 문제 발생 시 원인을 파악할 수 있음
