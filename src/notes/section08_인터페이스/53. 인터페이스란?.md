# TypeScript: 인터페이스 (Interface) — 필요성

## 문제 상황

같은 객체 타입이 여러 변수에 반복될 때.

```typescript
// ❌ 타입이 반복됨
const user1: { name: string; age: number; gender: string } = {
  name: "홍길동",
  age: 20,
  gender: "male",
};

const user2: { name: string; age: number; gender: string } = {
  name: "이철수",
  age: 25,
  gender: "male",
};

const user3: { name: string; age: number; gender: string } = {
  name: "김영희",
  age: 30,
  gender: "female",
};
```

### 문제점

- 속성이 바뀌면 모든 변수의 타입을 **일일이 수정**해야 함
- 코드 중복으로 **가독성 저하**
- **유지보수** 어려움

```typescript
// name → uname 으로 바꾸면 전부 다 수정해야 함 😱
const user1: { uname: string; ... } = { ... };
const user2: { uname: string; ... } = { ... };
const user3: { uname: string; ... } = { ... };
```

---

## 해결책 → 인터페이스

공통 객체 타입을 **한 곳에 정의**하고 재사용.

```typescript
// ✅ 인터페이스로 타입 한 번만 정의
interface User {
  name: string;
  age: number;
  gender: string;
}

const user1: User = { name: "홍길동", age: 20, gender: "male" };
const user2: User = { name: "이철수", age: 25, gender: "male" };
const user3: User = { name: "김영희", age: 30, gender: "female" };

// name → uname 변경 시 인터페이스만 수정하면 끝!
```

## 인터페이스 문법

```typescript
interface 식별자 {
  속성명: 타입;
  속성명: 타입;
}
```

```typescript
// User 인터페이스 정의
interface User {
  name: string;
  age: number;
  gender: string;
}

// 인터페이스를 타입으로 사용
const user1: User = { name: "홍길동", age: 20, gender: "male" };
const user2: User = { name: "이철수", age: 25, gender: "male" };
const user3: User = { name: "김영희", age: 30, gender: "female" };
```

---

## 인터페이스의 장점

### 유지보수성

```typescript
// name → uname 변경 시 인터페이스만 수정하면 모든 변수에 반영
interface User {
  uname: string; // 여기만 바꾸면 끝!
  age: number;
  gender: string;
}
```

### 가독성

```typescript
// ❌ 타입 반복 — 가독성 나쁨
const user1: { name: string; age: number; gender: string } = { ... };
const user2: { name: string; age: number; gender: string } = { ... };

// ✅ 인터페이스 사용 — 가독성 좋음
const user1: User = { ... };
const user2: User = { ... };
```

---

## 인터페이스 = 하나의 타입

```
number → 숫자 타입
string → 문자 타입
User   → name, age, gender 속성을 가진 객체 타입
```

> 인터페이스로 정의한 식별자는 `number`, `string`처럼 **하나의 타입으로 인식**됨

---

## 인터페이스 활용 팁

### 반복되지 않아도 인터페이스 사용 권장

```typescript
// 변수 하나여도 인터페이스 사용 OK (오히려 권장)
interface User {
  name: string;
  age: number;
}

const user: User = { name: "홍길동", age: 20 };
// → 변수만 봐도 타입을 바로 파악 가능
```

### 별도 파일로 분리 가능

```typescript
// types/user.ts
export interface User {
  name: string;
  age: number;
  gender: string;
}

// main.ts
import { User } from "./types/user";

const user: User = { name: "홍길동", age: 20, gender: "male" };
```

> 인터페이스를 별도 파일로 분리하면 **프로젝트 유지보수성 향상**  
> 반복 여부와 관계없이 객체 타입은 **인터페이스로 정의하는 것을 권장**
