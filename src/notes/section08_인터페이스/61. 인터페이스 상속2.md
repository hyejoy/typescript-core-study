# TypeScript: 인터페이스 상속 (extends)

## 상속이란?

부모 인터페이스의 속성을 자식 인터페이스가 **물려받는** 것.  
`extends` 키워드 사용.

```
상위집합(부모) → 하위집합(자식)이 물려받음
Person(사람) → Developer(개발자)는 사람이기도 함 → 상속 관계 성립
```

---

## 문법

```typescript
interface 자식인터페이스 extends 부모인터페이스 {
  // 추가 속성만 정의
}
```

---

## 예시

### 상속 전

```typescript
interface Person {
  name: string;
  age: number;
}

interface Developer {
  name: string; // Person과 중복
  age: number; // Person과 중복
  skill: string;
}
```

### 상속 후

```typescript
interface Person {
  name: string;
  age: number;
}

// Developer는 Person을 상속 → name, age 중복 제거
interface Developer extends Person {
  skill: string; // 추가 속성만 정의
}

// Developer는 name + age + skill 모두 포함
const dev: Developer = {
  name: "홍길동",
  age: 20,
  skill: "TypeScript",
};
```

---

## 주의사항 — 부모 변경 시 자식도 영향받음

```typescript
interface Person {
  name: string;
  age: string; // number → string 으로 변경
}

// Developer도 age가 string으로 바뀜
const dev: Developer = {
  name: "홍길동",
  age: 20, // ❌ 에러 — string이어야 함
  age: "20", // ✅
  skill: "TypeScript",
};
```

> 상속 관계는 **부모-자식이 밀접하게 연결**됨  
> 부모 타입 변경 시 자식에게도 영향이 전파됨

---

## 정리

|                | 설명                                             |
| -------------- | ------------------------------------------------ |
| 키워드         | `extends`                                        |
| 의미           | 부모 속성을 자식이 물려받음                      |
| 중복 속성      | 자식에서 제거 (부모에서 상속됨)                  |
| 부모 변경 영향 | 자식에게도 전파됨                                |
| 적용 기준      | 의미론적으로 부모-자식 관계일 때 (개발자 ⊂ 사람) |

---

## 추가 예시 — Car & BMW

```typescript
interface Car {
  name: string;
  color: string;
}

// BMW는 자동차의 한 종류 → 부모-자식 관계 성립
interface BMW extends Car {
  price: number;
}

const bmw: BMW = {
  name: "BMW",
  color: "black",
  price: 50000000,
};
```

---

## ⚠️ 상속 남용 주의 — 단순 중복 제거 목적으로 사용 금지

```typescript
interface Phone {
  name: string;
  color: string;
  // ...
}

interface TV {
  name: string;
  color: string;
  // ...
}

// ❌ 공통 속성이 있다고 상속 관계로 만들면 안 됨
interface TV extends Phone {
  // TV는 Phone이 아님!
  // ...
}
```

> **공통 속성이 있다 ≠ 상속 관계**  
> 상속은 의미론적으로 **"A는 B이기도 하다"** 관계일 때만 사용

| 올바른 상속                              | 잘못된 상속                       |
| ---------------------------------------- | --------------------------------- |
| Developer extends Person (개발자는 사람) | TV extends Phone (TV는 폰이 아님) |
| BMW extends Car (BMW는 자동차)           | 단순 속성 중복 제거 목적          |
