# TypeScript: 인덱스 시그니처 (Index Signature)

## 인덱스 시그니처란?

객체의 속성 타입을 **동적으로** 정의할 때 사용하는 문법.  
속성이 얼마나 추가될지 모를 때 유용.

---

## 문제 상황 — 속성 추가할 때마다 타입도 추가해야 함

```typescript
// ❌ 번거로운 방식 — 속성 추가할 때마다 타입에도 추가 필요
const user: {
  name: string;
  gender: string;
  address: string;
} = {
  name: "홍길동",
  gender: "male",
  address: "서울",
};
```

---

## 인덱스 시그니처 문법

```typescript
// { [키명: 키타입]: 값타입 }
const user: { [key: string]: string } = {
  name: "홍길동",
  gender: "male",
  address: "서울",
  // 문자열 키 + 문자열 값이면 얼마든지 추가 가능
};
```

→ "이 객체의 키는 string, 값도 string" 이라는 의미

---

## 값 타입이 다양할 때 — union 사용

```typescript
const user: {
  [key: string]: string | number | (() => string);
} = {
  name: "홍길동", // string ✅
  age: 20, // number ✅
  getName: () => "홍길동", // () => string ✅
};
```

> ⚠️ union에 함수 타입 포함 시 **소괄호로 묶어야 함**  
> `string | (() => string)` → 소괄호 없으면 에러

---

## 인덱스 시그니처 위반 — 지정한 타입과 다른 값

```typescript
const user: { [key: string]: string } = {
  name: "홍길동",
  age: 20, // ❌ 에러 — 값 타입이 string이어야 하는데 number
};
```

---

## 인덱스 시그니처의 타입 안전성 문제

### 문제 1 — `this.속성` 참조 시 타입 불명확

```typescript
const person: {
  [key: string]: string | (() => string);
  name: string; // 명시적으로 추가해야 함
} = {
  name: "홍길동",
  getName() {
    return this.name; // ❌ 에러 — name이 string인지 함수인지 불명확
    return "홍길동"; // ✅ 명시적 문자열은 OK
  },
};
```

인덱스 시그니처로 값이 `string | (() => string)` 이면  
`this.name`이 문자열인지 함수인지 TS가 확신 못함  
→ `name: string` 을 명시적으로 추가해야 해결

### 문제 2 — 함수 속성 호출 시 타입 불명확

```typescript
person.printName(); // ❌ 에러 — printName이 string일 수도 있어서 호출 불가
```

→ `printName: () => string` 을 명시적으로 추가해야 해결

### 결론

```typescript
// 결국 이렇게 일일이 다 명시해야 함
const person: {
  [key: string]: string | (() => string);
  name: string;
  printName: () => string;
} = { ... };
// → 인덱스 시그니처를 쓰는 의미가 없어짐
```

> 인덱스 시그니처는 자유도를 높이지만 **타입 안전성이 떨어짐**  
> 남용하면 오히려 타입을 일일이 다 명시해야 하는 상황이 생길 수 있음

---

## 일반 방식 vs 인덱스 시그니처 비교

|              | 일반 방식               | 인덱스 시그니처               |
| ------------ | ----------------------- | ----------------------------- |
| 속성 추가 시 | 타입에도 매번 추가 필요 | 자동으로 허용                 |
| 유연성       | 낮음                    | 높음                          |
| 타입 안전성  | 높음                    | 상대적으로 낮음               |
| 적합한 경우  | 속성이 고정된 객체      | 속성이 동적으로 추가되는 객체 |

---

## 인터페이스에서도 사용 가능

```typescript
// 인터페이스에 인덱스 시그니처 적용
interface User {
  [key: string]: string | number;
}

const user1: User = {
  name: "홍길동",
  age: 20, // number도 허용
  address: "서울",
};
```

> 인덱스 시그니처는 인터페이스 고유 문법이 아니라  
> **객체 타입을 지정할 때 어디서든 사용 가능**한 문법
