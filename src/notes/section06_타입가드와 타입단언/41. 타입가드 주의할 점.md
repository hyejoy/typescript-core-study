# 타입가드 주의할 점

## 문제 상황

유니온 타입의 모든 경우의 수를 if문으로 처리했음에도 타입 에러 발생

## 원인

반환값 타입이 명시되어 있으면 TypeScript는 매개변수 타입을 고려하지 않고
함수 내부에서 반드시 해당 타입을 반환하는지만 체크
→ 모든 if문이 실행되지 않으면 아무것도 반환하지 않는 경우의 수가 생김
→ 타입 에러 발생

## 해결 방법 4가지

### 1. 반환 타입에 undefined 추가

```typescript
function add(
  a: number | string,
  b: number | string,
): number | string | undefined {
  // ...
}
```

아무것도 반환하지 않으면 undefined를 반환하기 때문

### 2. 반환 타입 제거 (타입 추론에 맡기기)

```typescript
function add(a: number | string, b: number | string) {
  // 타입 추론이 자동으로 string | number | undefined로 추론
}
```

### 3. 모든 경우의 수에서 반환하게 처리

```typescript
function add(a: number | string, b: number | string): number | string {
  if (...) return a + b
  else if (...) return a + b
  else return String(a) + String(b)  // 반드시 반환
}
```

### 4. 마지막에 에러 던지기

```typescript
function add(a: number | string, b: number | string): number | string {
  if (...) return a + b
  else if (...) return a + b
  throw new Error('invalid parameter type')  // 모든 경우 처리됨으로 인식
}
```

## 정리

4가지 방법 중 어떤 게 더 좋다고 구분하기 어려움
→ 상황에 따라 적절한 방법 선택
