# TypeScript: 함수 오버로드 (Function Overload)

## 함수 오버로드란?

동일한 함수 이름으로 **서로 다른 매개변수/반환값 조합**을 여러 개 정의하는 기능.  
함수 시그니처를 케이스별로 **더 명확하게** 표현하고 싶을 때 사용.

> ⚠️ 필수가 아닌 **선택** — 사용하지 않아도 에러 없음

---

## 오버로드 없이 작성

```typescript
function greet(name: string, age?: number): string {
  if (age) return `Hello, ${name}! You are ${age} years old.`;
  return `Hello, ${name}!`;
}

greet("홍길동"); // ✅
greet("홍길동", 20); // ✅
```

→ 두 호출 모두 `(name: string, age?: number) => string` 으로 동일하게 보임

---

## 오버로드 시그니처 추가

함수 구현부 **위에** 케이스별 선언부만 작성 (구현부 없음).

```typescript
// 오버로드 시그니처 (케이스 1: name만)
function greet(name: string): string;

// 오버로드 시그니처 (케이스 2: name + age)
function greet(name: string, age: number): string;

// 실제 구현부 (오버로드 시그니처를 포괄하는 시그니처)
function greet(name: string, age?: number): string {
  if (age) return `Hello, ${name}! You are ${age} years old.`;
  return `Hello, ${name}!`;
}
```

> ⚠️ 오버로드 시그니처에서는 `?` (옵셔널) 사용 금지  
> → 옵셔널 쓰면 구현부와 동일해져서 케이스 구분 의미 없음

---

## 오버로드 사용 전 vs 후 비교

```typescript
// 오버로드 없음
greet("홍길동");
// 타입: (name: string, age?: number) => string  ← 항상 같음

// 오버로드 있음
greet("홍길동");
// 타입: (name: string) => string  ← 케이스에 맞게 다르게 보임

greet("홍길동", 20);
// 타입: (name: string, age: number) => string  ← 케이스에 맞게 다르게 보임
```

---

## 오버로드의 장점

| 장점             | 설명                                    |
| ---------------- | --------------------------------------- |
| 타입 명확성      | 호출 케이스별로 정확한 타입 표시        |
| 자동완성 지원    | 에디터에서 선택 가능한 케이스 목록 표시 |
| 타입 안전성 증가 | 잘못된 인자 조합 방지                   |

## 추가 예제 — union 타입과 함께 쓸 때 효과적

### 오버로드 없을 때 — 타입 추론이 부정확

```typescript
function add(n1: number | string, n2: number | string): number | string {
  if (typeof n1 === "number" && typeof n2 === "number") return n1 + n2;
  if (typeof n1 === "string" && typeof n2 === "string") return n1 + n2;
  throw new Error("invalid args");
}

const result = add(1, 2); // → number | string (부정확)
const result2 = add("a", "b"); // → number | string (부정확)
```

### 오버로드 시그니처 추가 — 케이스별 정확한 타입

```typescript
// 오버로드 시그니처
function add(n1: number, n2: number): number;
function add(n1: string, n2: string): string;

// 구현부
function add(n1: number | string, n2: number | string): number | string {
  if (typeof n1 === "number" && typeof n2 === "number") return n1 + n2;
  if (typeof n1 === "string" && typeof n2 === "string") return n1 + n2;
  throw new Error("invalid args");
}

const result = add(1, 2); // → number ✅
const result2 = add("a", "b"); // → string ✅
```

### 반환 타입이 입력에 따라 다른 경우 — 오버로드 3개

```typescript
// 오버로드 시그니처
function convert(value: string): string;
function convert(value: number): number;
function convert(value: boolean): string;

// 구현부
function convert(value: string | number | boolean): string | number {
  if (typeof value === "string") return value.toUpperCase();
  if (typeof value === "number") return value * 2;
  return String(value);
}

convert("hello"); // → string
convert(10); // → number
convert(true); // → string
```

## 정리

```
오버로드 시그니처 = 함수 선언부만 있고 구현부 없음 (여러 개 가능)
구현 시그니처    = 실제 함수 구현부 (오버로드 시그니처를 포괄해야 함)
```

### 오버로드가 특히 유용한 경우

- union 타입으로 매개변수/반환값을 지정했을 때 타입 추론이 부정확한 경우
- 매개변수 조합에 따라 반환 타입이 달라지는 경우

> 오버로드는 **필수 아님** — 하지만 타입 명확성과 자동완성을 위해 사용하면 좋음
