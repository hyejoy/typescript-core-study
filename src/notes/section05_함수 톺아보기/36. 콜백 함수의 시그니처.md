# TypeScript: 콜백 함수의 함수 시그니처

## 핵심 원칙

> 콜백 함수라고 특별한 방법이 있는 게 아님  
> **콜백 함수도 매개변수이므로, 그 함수의 시그니처를 타입으로 지정**하면 됨

---

## 기본 패턴

```typescript
// 콜백을 받는 함수의 매개변수 타입 = (매개변수: 타입) => 반환타입
function fetchData(callback: (data: string) => void): void {
  callback("hello");
}

// 콜백 함수 자체에도 시그니처 지정
fetchData((data: string): void => {
  console.log(data);
});
```

---

## 예시 1 — 단순 콜백

```typescript
function fetchData(callback: (data: string) => void): void {
  callback("fetched data");
}

fetchData((data: string): void => {
  console.log(data); // "fetched data"
});
```

---

## 예시 2 — 일반 매개변수 + 콜백 매개변수

```typescript
function processData(data: string, callback: (result: string) => void): void {
  const result = data.toUpperCase();
  callback(result);
}

processData("hello", (result: string): void => {
  console.log(result); // "HELLO"
});
```

---

## 예시 3 — 함수 참조를 콜백으로 전달

```typescript
function doubleNum(item: number): void {
  console.log(item * 2);
}

function powNum(item: number): void {
  console.log(item ** 2);
}

function processItems(items: number[], callback: (item: number) => void): void {
  items.forEach(callback);
}

processItems([1, 2, 3], doubleNum); // 2, 4, 6
processItems([1, 2, 3], powNum); // 1, 4, 9
```

---

## 예시 4 — Error | null 패턴 (Node.js 스타일)

```typescript
function fetchsData(
  callback: (error: Error | null, data: string | null) => void,
): void {
  setTimeout(() => {
    callback(null, "Data fetched successfully");
  }, 1000);
}

fetchsData((error: Error | null, data: string | null): void => {
  if (error) {
    console.error(error);
  } else {
    console.log(data);
  }
});
```

> `Error` 타입 → `new Error()`로 생성한 에러 객체의 타입  
> `Error | null` → 에러가 올 수도 있고 null이 올 수도 있음

---

## 콜백 함수 시그니처 작성 순서

```
1. 콜백 함수 자체의 시그니처 작성
   (data: string): void => { ... }

2. 콜백을 받는 함수의 매개변수 타입으로 등록
   function fn(callback: (data: string) => void) { ... }
```
