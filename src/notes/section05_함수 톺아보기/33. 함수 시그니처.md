# TypeScript: 함수 시그니처 (Function Signature)

## 함수 시그니처란?

함수의 **매개변수와 반환값의 타입을 지정**하는 것.  
앞에서 배운 함수 타입 지정 방법에 공식 명칭을 붙인 것.

---

## 함수 선언문에서의 함수 시그니처

```typescript
function 함수명(매개변수: 타입): 반환타입 { ... }
```

---

## 예시 모음

### 매개변수 없고 반환값 없음

```typescript
function printHello(): void {
  console.log("hello");
}
```

### 매개변수 있고 반환값 있음

```typescript
function add(n1: number, n2: number): number {
  return n1 + n2;
}

function greet(name: string): string {
  return `Hello, ${name}!`;
}
```

### 복잡한 매개변수 (배열)

```typescript
function findLongString(strings: string[]): string {
  return strings.reduce((longest, current) =>
    current.length > longest.length ? current : longest,
  );
}

findLongString(["apple", "watermelon", "kiwi"]); // → "watermelon"
```

> 매개변수 타입을 지정하면 함수 내부의 타입도 함께 추론됨  
> `strings: string[]` → 배열 요소 접근 시 `string`으로 추론 → 내부 에러 해결

---

## 핵심 원칙

> 함수가 복잡해지더라도 본질은 같음  
> **매개변수 타입** + **반환값 타입** 을 지정하는 것이 함수 시그니처의 전부

| 요소          | 위치      | 예시                             |
| ------------- | --------- | -------------------------------- |
| 매개변수 타입 | 소괄호 안 | `(n1: number, n2: number)`       |
| 반환값 타입   | 소괄호 뒤 | `: number`, `: string`, `: void` |
