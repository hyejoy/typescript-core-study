# TypeScript: 콜백 함수 호출 시 타입 생략 가능 여부

## 핵심

함수의 **매개변수 쪽에 타입을 지정**해두면, 호출 시 직접 작성하는 콜백 함수는 **타입 추론**이 되어 타입 생략 가능.

---

## 인라인 콜백 — 타입 생략 가능 ✅

```typescript
function processData(data: string, callback: (result: string) => void): void {
  callback(data.toUpperCase());
}

// 타입 명시
processData("hello", (result: string): void => {
  console.log(result);
});

// 타입 생략 → 매개변수 타입에서 추론됨, 에러 없음
processData("hello", (result) => {
  console.log(result);
});
```

---

## 외부 선언 함수를 참조값으로 전달 — 타입 생략 불가 ❌

```typescript
// 외부에 선언된 함수 → 타입 직접 지정 필요
function doubleNum(item: number): void {
  console.log(item * 2);
}

function processItems(items: number[], callback: (item: number) => void): void {
  items.forEach(callback);
}

// ✅ 타입 있음
processItems([1, 2, 3], doubleNum);

// 만약 doubleNum에서 타입 제거하면
function doubleNum(item) { ... } // ❌ 에러 발생
```

> 외부 함수는 호출 매개변수의 타입 추론을 받지 못하므로 **직접 타입 지정 필수**

---

## 타입 명시 vs 생략 장단점

|             | 타입 생략                                    | 타입 명시        |
| ----------- | -------------------------------------------- | ---------------- |
| 코드 길이   | 짧고 간결                                    | 길어짐           |
| 타입 명확성 | 낮음                                         | 높음             |
| 협업 가독성 | 함수 정의와 호출이 다른 파일이면 파악 어려움 | 빠르게 파악 가능 |
| 유지보수    | 타입 추론에 의존                             | 명확한 타입 보장 |

---

## 권장 방향

> 함수 정의부와 호출부가 **다른 파일**에 있는 경우가 많은 실무에서는  
> **호출 시에도 타입을 명시하는 것을 권장**

```typescript
// 권장
processData("hello", (result: string): void => {
  console.log(result);
});
```
