# TypeScript: 함수 표현식의 함수 시그니처

## 함수 시그니처란?

함수의 **매개변수와 반환값의 타입을 지정**하는 것.

함수 표현식에서는 **두 가지 방법** 중 하나를 선택해서 작성.

---

## 방법 1 — 함수에 직접 타입 지정 (선언문 방식과 동일)

```typescript
// 매개변수 없고 반환값 없음
const printHello = function (): void {
  console.log("hello");
};

// 두 수의 합 반환
const add = function (n1: number, n2: number): number {
  return n1 + n2;
};

// 문자열 매개변수, 문자열 반환
const greet = function (name: string): string {
  return `Hello, ${name}!`;
};

// 문자열 배열 받아서 가장 긴 문자열 반환
const findLongString = function (strings: string[]): string {
  return strings.reduce((acc, cur) => (cur.length > acc.length ? cur : acc));
};
```

---

## 방법 2 — 변수에 타입 지정 (화살표 함수 형태)

```typescript
// 매개변수 없고 반환값 없음
const printHello: () => void = function () {
  console.log("hello");
};

// 두 수의 합 반환
const add: (n1: number, n2: number) => number = function (n1, n2) {
  return n1 + n2;
};

// 문자열 매개변수, 문자열 반환
const greet: (name: string) => string = function (name) {
  return `Hello, ${name}!`;
};

// 문자열 배열 받아서 가장 긴 문자열 반환
const findLongString: (strings: string[]) => string = function (strings) {
  return strings.reduce((acc, cur) => (cur.length > acc.length ? cur : acc));
};
```

---

## 두 방법 비교

|           | 방법 1 (함수에 직접)    | 방법 2 (변수에 지정)         |
| --------- | ----------------------- | ---------------------------- |
| 타입 위치 | 매개변수 옆 + `)` 뒤    | 변수명 뒤 (화살표 함수 형태) |
| 가독성    | 함수 내부에서 타입 확인 | 변수 선언부에서 타입 확인    |
| 권장 여부 | ✅ 둘 다 OK             | ✅ 둘 다 OK                  |

> ⚠️ 두 방법을 **동시에 사용하는 것은 비권장** — 중복이므로 하나만 선택

---

## 화살표 함수도 동일

```typescript
// 방법 1
const add = (n1: number, n2: number): number => n1 + n2;

// 방법 2
const add: (n1: number, n2: number) => number = (n1, n2) => n1 + n2;
```
