# TypeScript: 타입 추론 주의사항

## 핵심 원칙

> **타입 추론을 제대로 활용하려면 변수 선언과 동시에 값을 할당해야 함**

---

## 선언만 하고 나중에 할당하면 → `any`

```typescript
// ❌ 선언만 하고 나중에 할당
let a; // → any로 추론
a = 10; // 숫자를 넣었지만 여전히 any
a = "hello"; // ✅ 문자도 되고
a = true; // ✅ 논리값도 됨 → 타입 안전성 없음

// ✅ 선언과 동시에 할당
let a = 10; // → number로 추론
a = "hello"; // ❌ 에러 — number에 string 할당 불가
```

---

## 함수 매개변수도 같은 맥락

```typescript
// ❌ 초기값 없음 → any 추론 → 에러
function add(n1, n2) {
  // 매개변수 → any
  return n1 + n2; // 반환값 → any
}

// ✅ 초기값(기본값) 지정 → 타입 추론 가능
function add(n1 = 0, n2 = 0) {
  // → number로 추론
  return n1 + n2; // → number로 추론
}
```

---

## 선언과 동시에 할당해도 `any`인 경우 → 명시적 타입 지정 필요

```typescript
// API 응답값 → 어떤 값인지 알 수 없음 → any로 추론
const result = await fetch("/api/user").then(res => res.json()); // → any

// ✅ 이럴 때는 명시적으로 타입 지정
const result: { name: string; age: number } = await fetch(...);
```

---

## 정리

| 상황                           | 추론 결과                   |
| ------------------------------ | --------------------------- |
| 선언 + 즉시 할당 (기본 자료형) | ✅ 정확한 타입 추론         |
| 선언만 하고 나중에 할당        | ❌ `any`                    |
| 함수 매개변수 (초기값 없음)    | ❌ `any`                    |
| 함수 매개변수 (초기값 있음)    | ✅ 정확한 타입 추론         |
| API 응답값                     | ❌ `any` → 명시적 지정 필요 |
