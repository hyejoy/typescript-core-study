# TypeScript: `undefined` · `null` 타입 추론 주의사항

## undefined와 null이 any로 추론되는 이유

JS에서 `null`과 `undefined`는 **나중에 다른 값이 들어올 가능성**을 내포하는 값.

| 값          | JS에서의 의미                                                 |
| ----------- | ------------------------------------------------------------- |
| `null`      | 개발자가 **의도적으로** 값을 비워둠 → 나중에 채울 가능성 있음 |
| `undefined` | 값이 **아직 할당되지 않은** 상태 → 나중에 할당될 가능성 있음  |

```typescript
let a = undefined; // any로 추론
let b = null; // any로 추론

a = 10; // ✅ any라서 가능
b = "hello"; // ✅ any라서 가능
```

→ TS 타입 추론이 `any`로 추론한 건 **"나중에 다른 값이 들어올 수 있다"** 는 점을 고려한 것

---

## 문제점

```typescript
// 명시적으로 타입 지정 시
let a: undefined = undefined;
let b: null = null;

a = 10; // ❌ 에러 — undefined 타입에 number 할당 불가
b = "hello"; // ❌ 에러 — null 타입에 string 할당 불가
```

→ 나중에 다른 값을 할당할 수 없게 됨

---

## 해결책 — union 타입 사용

```typescript
// 나중에 들어올 타입을 union으로 함께 지정
let a: undefined | number = undefined;
a = 10; // ✅

let b: null | string = null;
b = "hello"; // ✅
```

---

## 결론

> `undefined`와 `null`은 타입 추론에 의존하지 말고  
> **명시적으로 union 타입으로 지정**하는 것이 좋음

```typescript
// ❌ 타입 추론에 의존
let a = undefined; // any → 타입 불명확

// ✅ 명시적 union 타입 지정
let a: undefined | number = undefined; // 의도가 명확
```
