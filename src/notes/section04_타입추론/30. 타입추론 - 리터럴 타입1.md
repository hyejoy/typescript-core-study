# TypeScript: 리터럴 타입 (Literal Type)

## 리터럴 타입이란?

**특정 값 자체를 타입으로 사용**하는 기능.  
포괄적인 타입(`string`, `number`) 대신 정확한 값을 타입으로 지정.

```typescript
// string → 모든 문자열 허용
const gender: string = "male";
gender = "a"; // ✅ 어떤 문자열도 할당 가능

// 리터럴 타입 → "male"만 허용
const gender: "male" = "male";
gender = "a"; // ❌ 에러
gender = "female"; // ❌ 에러
gender = "male"; // ✅
```

---

## union과 함께 사용

```typescript
// "male" 또는 "female"만 허용
const gender: "male" | "female" = "male";
gender = "female"; // ✅
gender = "a"; // ❌ 에러
```

---

## 다양한 리터럴 타입

Symbol을 제외한 모든 값이 리터럴 타입 가능.

```typescript
// 문자열 리터럴
const direction: "left" | "right" | "up" | "down" = "left";

// 숫자 리터럴
const statusCode: 200 | 400 | 404 | 500 = 200;

// 논리 리터럴
const flag: true = true;

// 객체 리터럴
const user: { name: "철수" } = { name: "철수" };
user.name = "영희"; // ❌ 에러 — "철수"만 허용
```

---

## string vs 리터럴 타입 비교

|             | `string`    | `"male" \| "female"`          |
| ----------- | ----------- | ----------------------------- |
| 허용 값     | 모든 문자열 | "male" 또는 "female"만        |
| 타입 안전성 | 낮음        | 높음                          |
| 사용 예     | 일반 문자열 | 정해진 값 중 하나만 허용할 때 |

---

## 활용 예시

```typescript
// API 상태 코드
type Status = "success" | "error" | "loading";

// 방향 제한
type Direction = "left" | "right" | "up" | "down";

// 사용자 역할
type Role = "admin" | "user" | "guest";
```

> 리터럴 타입은 특정 값만 허용하는 변수를 만들 때 사용  
> union 타입과 함께 자주 쓰임
