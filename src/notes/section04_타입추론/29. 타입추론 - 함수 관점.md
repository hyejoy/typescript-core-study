# TypeScript: 함수의 타입 추론

## 함수 타입 추론 2가지 관점

| 관점     | 타입 추론 여부                          |
| -------- | --------------------------------------- |
| 매개변수 | ❌ 항상 `any`로 추론 → 반드시 명시 필요 |
| 반환값   | 상황에 따라 가능 / 불가능               |

---

## 매개변수 — 항상 명시해야 함

```typescript
// ❌ 타입 미지정 → any로 추론 → 에러 발생
function add(n1, n2) {
  // 에러: 매개변수 n1이 암시적으로 any 타입
  return n1 + n2;
}

// ✅ 명시적 타입 지정
function add(n1: number, n2: number) {
  return n1 + n2;
}
```

> 매개변수로 전달될 값은 너무 다양하기 때문에 TS가 추론 불가 → **항상 직접 명시**

---

## 반환값 — 상황에 따라 다름

### 추론 가능한 경우

```typescript
// 매개변수 타입이 명확하면 반환값도 추론 가능
function add(n1: number, n2: number) {
  return n1 + n2; // → number로 추론
}

// 직접 값을 반환하는 경우
function greeting() {
  return "hello"; // → string으로 추론
}
```

### 추론 불가능한 경우 → `any`

```typescript
// 매개변수 타입 미지정 → 반환값도 any
function add(n1, n2) {
  return n1 + n2; // → any로 추론
}

// API 결과 등 타입을 알 수 없는 값 반환 시 → any
```

---

## 흐름 정리

```
매개변수 타입 미지정
  → 매개변수 any
  → 반환값도 any (매개변수 기반 연산이라면)

매개변수 타입 지정 (number)
  → 반환값 추론 가능 (number + number = number)
  → 반환값 자동으로 number 추론
```

---

## 결론

> - 매개변수 타입 → **항상 명시**
> - 반환값 타입 → 추론 가능하면 생략 가능, 불확실하면 명시 권장
