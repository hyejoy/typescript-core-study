# TypeScript: `const`와 리터럴 타입 추론

## let vs const 타입 추론 차이

```typescript
// let → 재할당 가능 → 포괄적 타입으로 추론
let num = 10; // → number
let str = "hello"; // → string

// const → 재할당 불가 → 리터럴 타입으로 추론
const num = 10; // → 10 (리터럴)
const str = "hello"; // → "hello" (리터럴)
```

---

## 기본 자료형 — let vs const 비교

| 선언                | 값   | 추론 타입       |
| ------------------- | ---- | --------------- |
| `let num = 10`      | 10   | `number`        |
| `const num = 10`    | 10   | `10` (리터럴)   |
| `let str = "hi"`    | "hi" | `string`        |
| `const str = "hi"`  | "hi" | `"hi"` (리터럴) |
| `let bool = true`   | true | `boolean`       |
| `const bool = true` | true | `true` (리터럴) |

> Symbol 제외한 모든 기본 자료형에 해당

---

## 참조 자료형 — const여도 리터럴 추론 안 됨

```typescript
// const여도 포괄적 타입으로 추론
const arr = [1, 2, 3]; // → number[] (리터럴 아님)
const obj = { name: "철수" }; // → { name: string } (리터럴 아님)
```

### 이유

```typescript
const arr = [1, 2, 3];
arr.push(4); // ✅ 재할당은 안 되지만 내부 값은 변경 가능

const obj = { name: "철수" };
obj.name = "영희"; // ✅ 참조를 통해 내부 값 변경 가능
```

`const`는 **재할당 불가**이지, **내부 값 변경 불가**가 아님  
→ 내부 값이 바뀔 수 있으므로 리터럴 타입으로 추론 불가

---

## 정리

|         | 기본 자료형                           | 참조 자료형                   |
| ------- | ------------------------------------- | ----------------------------- |
| `let`   | 포괄적 타입 (`number`, `string` ...)  | 포괄적 타입                   |
| `const` | **리터럴 타입** (`10`, `"hello"` ...) | 포괄적 타입 (변경 가능하므로) |
