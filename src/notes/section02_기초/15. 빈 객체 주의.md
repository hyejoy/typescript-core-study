# TypeScript: 빈 객체 타입 주의사항

## 빈 객체 타입 `{}`의 함정

```typescript
const obj: {} = {}; // ⚠️ 진정한 빈 객체 타입이 아님
```

`{}`으로 타입을 지정하면 **undefined와 null을 제외한 모든 값**을 허용함.

```typescript
const obj: {} = {};

obj = undefined; // ❌ 에러
obj = null; // ❌ 에러

obj = 30; // ✅ 에러 없음 (숫자인데도!)
obj = "hello"; // ✅ 에러 없음
obj = [1, 2, 3]; // ✅ 에러 없음
obj = () => {}; // ✅ 에러 없음
```

---

## 왜 이런 현상이 발생하나?

JavaScript의 근본적인 특성 때문.

```
JS에서 모든 값은 객체 프로토타입의 메서드를 상속받음
→ 숫자도 toString(), valueOf() 같은 메서드를 가짐
→ "어떤 속성도 가지지 않는 객체 형태"인 {} 를 모든 값이 만족함
→ 결국 {} 타입 = undefined/null 제외한 모든 값 허용
```

---

## 진정한 빈 객체 타입 → `Record<string, never>`

```typescript
// ✅ 진정한 빈 객체 타입
const obj: Record<string, never> = {};

obj = 30; // ❌ 에러
obj = "hello"; // ❌ 에러
obj = [1, 2, 3]; // ❌ 에러
obj = { name: "홍" }; // ❌ 에러
obj = {}; // ✅ 빈 객체만 허용
```

---

## 정리

| 타입                    | 허용 범위                          |
| ----------------------- | ---------------------------------- |
| `{}`                    | undefined, null 제외한 **모든 값** |
| `Record<string, never>` | **빈 객체만** 허용                 |

> `Record`는 유틸리티 타입으로 나중에 자세히 배움  
> 지금은 **빈 객체 타입을 제대로 지정하려면 `Record<string, never>` 사용** 이것만 암기
